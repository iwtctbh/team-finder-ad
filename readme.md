# TeamFinder - Платформа для поиска команды для pet-проектов

## О проекте

TeamFinder - это веб-приложение, где разработчики, дизайнеры и другие специалисты могут находить единомышленников для совместной работы над pet-проектами. Зарегистрированные пользователи могут размещать идеи проектов, находить команду и откликаться на опубликованные предложения.

**Вариант реализации:** Вариант 3 - "Необходимые навыки" проектов, фильтрация проектов по навыкам.

## Функциональность

### Для всех пользователей (гостей)
- Просмотр списка проектов с пагинацией (12 проектов на странице)
- Просмотр страницы отдельного проекта
- Просмотр страниц пользователей
- Регистрация и вход в систему

### Для авторизованных пользователей
- Всё, что доступно гостям
- Создание, редактирование и завершение своих проектов
- Участие в проектах других пользователей
- Редактирование своего профиля (аватар, описание, контакты)
- Управление навыками своих проектов (добавление/удаление)
- Фильтрация проектов по необходимым навыкам
- Смена пароля
- Выход из системы

### Для администратора
- Все права авторизованных пользователей
- Управление пользователями и проектами через админ-панель
- Изменение пароля любого пользователя
- Блокировка и удаление аккаунтов

## Технологии

- Python 3.9+
- Django 4.2
- PostgreSQL 15
- Docker / Docker Compose
- HTML/CSS (готовые шаблоны)
- JavaScript (готовые скрипты)

## Установка и запуск

```bash
# 1. Клонирование репозитория
git clone https://github.com/iwtctbh/team-finder-ad.git
cd team-finder-ad

# 2. Создание и активация виртуального окружения
python -m venv venv
source venv/bin/activate  # Linux/Mac
source venv\Scripts\activate  # Windows

# 3. Установка зависимостей
pip install -r requirements.txt

# 4. Настройка .env (скопировать и заполнить)
cp .env_example .env
# Сгенерировать секретный ключ:
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
# Вставить ключ в .env, также указать TASK_VERSION=3

# 5. Запуск PostgreSQL через Docker
docker-compose up -d
docker ps

# 6. Миграции и создание суперпользователя
python manage.py migrate
python manage.py createsuperuser

# 7. Запуск сервера
python manage.py runserver
